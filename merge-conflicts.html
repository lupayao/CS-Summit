Merge Conflicts

Understanding Merge Conflicts Intermediate

Ready to Learn Module 3 of 4

Merge conflicts occur when Git can't automatically combine changes from different branches. While they might seem scary at first, they're actually Git's way of being careful and asking for your help to make important decisions about your code.

ü§ù

Think of it like group projects!

Imagine two students editing the same paragraph of a report at the same time. When they try to combine their work, they need to sit down together and decide which changes to keep, which to modify, and how to blend their ideas. That's exactly what happens with merge conflicts in Git!

Teaching Strategy

Frame merge conflicts as collaboration opportunities, not errors. When students encounter them, celebrate! It means multiple people are contributing to the project. Use the analogy of editors working on a newspaper - sometimes they need to discuss and merge their articles thoughtfully.

When Do Merge Conflicts Happen?

1
Different Branches

Two people work on different branches and modify the same lines in the same file.

2
Merge Attempt

When trying to merge the branches, Git finds conflicting changes and doesn't know which version to keep.

3
Git Pauses

Git stops the merge process and marks the conflicting areas in your files for manual resolution.

What Does a Merge Conflict Look Like?

When Git encounters a conflict, it marks the conflicting sections in your file with special symbols. Here's what you'll see:

index.html - Merge Conflict
<html lang="en">
<head>
  <title>My Awesome Website</title>
</head>
<body>
<<<<<<< HEAD
  <h1>Welcome to Our Class Project!</h1>
  <p>This is our amazing 7th grade website.</p>
=======
  <h1>Class Project Hub</h1>
  <p>Welcome to our collaborative coding space!</p>
>>>>>>> feature/new-header
</body>
</html>

Resolving Conflicts Step-by-Step

1
Find the Conflicts

Look for files marked with conflict indicators in your VS Code or Git interface. They'll be highlighted in red or with warning symbols.

2
Edit the File

Open the conflicted file and decide what the final version should look like. You can keep one version, combine both, or write something completely new.

3
Remove Markers

Delete all the conflict markers (<<<<<<<, =======, >>>>>>>) from your file. Leave only the final content you want.

4
Stage Changes

Add the resolved file to staging. This tells Git that you've fixed the conflict and the file is ready to commit.

5
Commit the Resolution

Commit your changes with a message like "Resolve merge conflict in index.html" to complete the merge.

Prevention Strategies

While conflicts are normal, here are some strategies to minimize them and make collaboration smoother:

üìã
Plan File Ownership

Assign different files or sections to different students. Create a "Who's Working on What" document.

üîÑ
Frequent Pulls

Encourage students to pull changes frequently to stay up-to-date with their teammates' work.

üí¨
Communication

Use clear commit messages and communicate about major changes before making them.

Command Line Equivalents

For advanced students or troubleshooting, here are the command line tools for handling merge conflicts:

Check conflict status: git status

See conflicted files: git diff --name-only --diff-filter=U

Abort merge (start over): git merge --abort

After resolving conflicts: git add [filename]

Complete the merge: git commit

Use merge tool: git mergetool

Pro Student Tip

Don't panic when you see a merge conflict! Take a deep breath, read the conflict carefully, and think about what the final result should look like. You can always ask for help or use git merge --abort to start over if you get stuck.

Building Confidence

Create a "safe space" for conflicts in your classroom. Set up practice repositories where students can experiment with creating and resolving conflicts without fear of breaking important code. Celebrate successful conflict resolutions as team achievements! Consider having students work in pairs initially, so they can support each other through the process.
