Branching & Merging

Branching and Merging

In Progress Module 2 of 4

Branching allows you to develop features, fix bugs, or experiment with new ideas in isolation from the main codebase. In this module, you'll learn how to create branches, switch between them, and merge changes.

Teacher Tip

Branching is a powerful concept to introduce to students working on group projects. It lets them work on features independently without stepping on each other's toes.

Visualizing Branching

Key Branching Concepts

Creating Branches

A branch is a lightweight movable pointer to a commit. When you create a branch, you're creating a new pointer to the current commit.

# In VSCode
1. Click the branch name in status bar
2. Select "Create new branch..."
3. Enter branch name and press Enter
Switching Branches

Switching branches updates the files in your working directory to match the version stored in that branch.

# In VSCode
1. Click the branch name in status bar
2. Select the branch you want to switch to
Merging Branches

Merging combines the changes from one branch into another. This is how you integrate work from different branches.

# In VSCode
1. Switch to the target branch (e.g., main)
2. Open Source Control panel
3. Click "..." (More Actions)
4. Select "Branch > Merge Branch..."
5. Choose the branch to merge from
Deleting Branches

Once a branch is merged, you can safely delete it to keep your repository clean.

# In VSCode
1. Open Source Control panel
2. Click "..." (More Actions)
3. Select "Branch > Delete Branch..."
4. Choose the branch to delete

Common Branching Strategies

Feature Branch Workflow

A simple and effective workflow for classroom projects:

  1. Create a new branch for each feature or bug fix
  2. Do all work related to that feature in the branch
  3. When complete, merge the branch back to main
  4. Delete the feature branch after merging
Student Tip

Always make sure you're on the right branch before starting work. Check the branch name in VSCode's status bar at the bottom of the window.

Branching Best Practices for Classroom Projects

Naming Conventions

Establish clear branch naming conventions to keep projects organized:

  • feature/ - For new features (e.g., feature/login-page)
  • bugfix/ - For bug fixes (e.g., bugfix/navigation-error)
  • hotfix/ - For urgent fixes (e.g., hotfix/security-issue)
  • username/ - For personal branches (e.g., jane/experiment)

Classroom Management

Tips for managing student repositories:

  • Protect the main branch to prevent direct commits
  • Require students to work in their own branches
  • Use pull requests for code review before merging
  • Create staging branches for testing integrations
Assessment Idea

Create a "broken" repository with multiple branches and ask students to identify which branches need to be merged and in what order to create a working application. This tests their understanding of both Git branching and code integration.
